<feature>
  <name>Config-driven channel router</name>
  <files>app/router.py, tests/test_router.py</files>
  <behavior>
    resolve_channel(transcript, config) -> str

    Config shape (from YAML):
      default_channel: "CDEFAULT"
      rules:
        - match_field: "organizer_email"
          pattern: "@eng.example.com"
          channel: "CENG001"
        - match_field: "title"
          pattern: "design review"
          channel: "CDESIGN1"

    Cases:
      organizer_email matching rule pattern -> rule channel
      title matching rule pattern (case-insensitive) -> rule channel
      no rule matches -> default_channel value
      empty rules list -> default_channel value
      config has no default_channel and no match -> ""
      first matching rule wins (not last)

    load_routing_config(path: str) -> dict
      Reads and yaml.safe_load the file at path.
      Raises FileNotFoundError if path does not exist (caller handles).
  </behavior>
  <implementation>
    - Use re.search(pattern, value, re.IGNORECASE) for matching (handles both substring and regex patterns)
    - match_field values supported: "title", "organizer_email"
    - Unknown match_field values are skipped (no crash)
    - transcript fields accessed with .get() to avoid KeyError on missing fields
    - load_routing_config uses yaml.safe_load (PyYAML already pinned)
    - No global state in router.py — functions are pure (config is passed in, not read inside resolve_channel)
  </implementation>
</feature>
