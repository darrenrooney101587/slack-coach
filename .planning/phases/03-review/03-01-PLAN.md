<feature>
  <name>Held-recap file state (review.py)</name>
  <files>app/review.py, tests/test_review.py</files>
  <behavior>
    hold_recap(blocks, channel_id, state_dir) -> str
      - Creates state_dir if it does not exist
      - Reads existing held_recaps.json (empty dict if missing or unreadable)
      - Generates a UUID recap_id
      - Stores {"blocks": blocks, "channel_id": channel_id, "held_at": int(time.time())} under recap_id
      - Writes updated dict back to held_recaps.json
      - Returns recap_id

    pop_recap(recap_id, state_dir) -> dict | None
      - Reads held_recaps.json
      - If recap_id found: removes it, writes updated file, returns the entry dict
      - If recap_id not found: returns None without modifying the file
      - Second call with same recap_id: returns None (idempotent by design)

    Test cases (input -> expected output):
      hold_recap(blocks, "C123", tmpdir) -> returns string UUID, file contains that UUID key
      hold_recap x2 same dir -> file contains two distinct UUID keys
      pop_recap(existing_id, tmpdir) -> returns entry dict, file no longer contains that key
      pop_recap(unknown_id, tmpdir) -> returns None, no exception
      pop_recap(already_popped_id, tmpdir) -> returns None (idempotent)
      hold_recap with corrupt json file -> overwrites gracefully, returns valid recap_id
  </behavior>
  <implementation>
    File: app/review.py
    Pattern: Follow votes.py _load/_save helpers — try/except on json.load, return {} on failure.
    Use uuid.uuid4() for recap_id.
    Single JSON file named "held_recaps.json" at state_dir root.
    No external dependencies — json, os, time, uuid are all stdlib.
    Do not add a logger (votes.py has one but the review module is simpler; keep it minimal).
  </implementation>
</feature>
