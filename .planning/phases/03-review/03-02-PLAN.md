  <action>
    Add send_review_dm(recap_id: str, blocks: list, channel_id: str, reviewer_user_id: str, bot_token: str) -> None to app/slack.py beneath post_recap.

    Implementation:
    - Build review_blocks = list(blocks) + [divider, section, actions_block]
    - Section text: f"*Review required* — this recap is held for <#{channel_id}>"
    - Actions block: block_id="review_actions", two buttons:
        Approve: action_id="recap_approve", value=recap_id, style="primary", text="Approve"
        Skip:    action_id="recap_skip",    value=recap_id, style="danger",  text="Skip"
    - Post via requests.post to SLACK_POST_URL (already defined in module) with:
        headers = {"Authorization": f"Bearer {bot_token}", "Content-Type": "application/json"}
        json = {"channel": reviewer_user_id, "blocks": review_blocks, "text": "Recap pending review"}
        timeout=10
    - Call response.raise_for_status() then check response.json()["ok"]
    - If ok is falsy, raise RuntimeError(f"Slack DM failed: {data.get('error')}")
    - Do not use conversations.open — passing reviewer_user_id as channel opens the DM automatically (verified in research)
    - Keep SLACK_POST_URL constant at module level (it is already there, reuse it)
  </action>
  <verify>poetry run python -c "from slack import send_review_dm; print('ok')"</verify>
  <done>send_review_dm importable from slack module, no syntax errors</done>
